# 자동매매 전략 개발2

이 폴더는 프로젝트 "돈버는 차트 전략 찾기"의 2차 인수인계용 아카이브다.
목적은 사람이 읽기 좋은 보고서가 아니라, 이후 어시스턴트가 즉시 맥락을 복원하고 같은 실수를 반복하지 않도록 하는 것이다.

## 현재 결론
- 아직 신뢰 가능한 최종 채택 전략은 없다.
- 과거 일부 플러스 결과는 백테스트 엔진 불일치/버그 가능성 때문에 무효 처리해야 한다.
- 가장 중요한 우선순위는 전략 탐색보다 백테스트 엔진의 재현성과 무결성 확보다.

## 폴더 구성
- 00_PROJECT_BRIEF.md: 프로젝트 목적, 제약, 철학
- 01_NON_NEGOTIABLES.md: 절대 제약과 사용자 작업 규칙
- 02_FAILED_FAMILIES.md: 이미 실패한 전략 축 정리
- 03_TIMELINE_AND_RESULTS.md: 실제 시도 순서와 요약 결과
- 04_ENGINE_INTEGRITY.md: 엔진 오류, 무효 처리된 결과, 재검증 규칙
- 05_NEXT_LOOP_PROTOCOL.md: 다음 어시스턴트가 따라야 할 고정 루프
- 06_ARTIFACTS_INDEX.md: 업로드된 CSV/코드 산출물 안내
- results/: 중간 결과 CSV 모음
- code/: 백테스트/그리드 코드

## 가장 중요한 행동 규칙
1. 엔진 검증 전에는 어떤 전략도 플러스라고 보고하지 말 것.
2. 같은 전략명은 완전히 동일한 규칙일 때만 재사용할 것.
3. 원시 사건 재스캔 → 후보 전략화 → 변형 5개 → 실패 시 재스캔 루프만 반복할 것.
4. 사용자는 중간 실패 보고를 싫어하므로, 최종적으로 의미 있는 전략 카드가 나왔을 때만 길게 보고할 것.
