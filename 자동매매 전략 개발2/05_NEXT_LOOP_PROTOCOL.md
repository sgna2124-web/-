# 05_NEXT_LOOP_PROTOCOL

다음 어시스턴트는 아래 루프만 반복한다.

## 0단계: 엔진 검증
- 1종목 소구간으로 SL/TP 처리 확인
- 수수료 적용 확인
- 최대 단일 트레이드 수익률 sanity check
- 같은 입력으로 2회 재실행 후 결과 일치 확인
- 이 단계 통과 전에는 어떤 전략 결과도 보고 금지

## 1단계: 원시 사건 재스캔
- 50개 안팎 사건 생성
- 사건 생성 축은 이전 실패 축과 겹치지 않게 다양화
- 계산 항목: 발생수, 3봉 평균, 5봉 평균, +0.3% 도달, -0.3% 도달, Edge, MFE-MAE

## 2단계: 후보 필터
최소 권고 필터
- Edge 1%p 미만 제외
- MFE-MAE 음수 제외
- Peak 가능성이 거의 없는 사건 제외
- 발생 수가 너무 과도하거나 너무 적은 사건 제외

## 3단계: 전략화
- 상위 3~5개만 전략화
- 롱/숏 승률 분리 기록
- 최종 자산 / Peak / MDD / 거래수 / 최대 단일 트레이드 필수

## 4단계: 변형
- 좋은 전략만 5개 이하의 정제형 변형 테스트
- 조건 누적/과최적화 금지
- 정제 방향: strict break, fast fail, body domination, compression entry, midpoint buffer 등

## 5단계: 판단
- 좋은 결과면 그 전략군에서 추가 정제
- 안 좋으면 즉시 폐기하고 다시 원시 사건 재스캔

## 보고 규칙
- 중간 실패 자세한 보고 금지
- 전략 카드로 부를 만한 결과가 나왔을 때만 길게 보고
- 사용자는 불필요한 설명보다 실행과 결론을 원함
