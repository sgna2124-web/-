가드레일 및 보고 규칙

사용자 요구 가드레일
1. 파라미터 미세조정 위주 실험은 지양한다.
2. 새로운 기준, 새로운 철학, 완전히 다른 전략처럼 보이는 시도를 선호한다.
3. MDD 10% 미만 여부가 매우 중요하다.
4. 현재는 롱 전용, 숏 전용을 따로 개발한다.
5. 나중에 포트폴리오처럼 결합한다.
6. topN, top200 같은 강한 순위 기반 필터는 신뢰하지 않는다.

보고 규칙
- 결과를 사용자에게 전달할 때는 기본적으로 아래 항목을 같이 말한다.
  최대 수익률
  MDD
  최대 보유 포지션
  승률
  cd_value
- 다만 최근 master_summary.csv에는 cd_value가 없을 수 있으므로, 이 경우에는 정확히 “이번 summary에는 cd_value 컬럼이 없다”라고 말하고, 필요 시 별도 계산 또는 스크립트 수정 계획을 제시한다.
- PF가 있으면 함께 언급하는 편이 좋다.

현재 코드/실험 해석 시 주의할 점
1. v42는 잘못된 레버리지 모델이었다.
- position_fraction 0.10으로 인해 총노출이 왜곡되었다.
- v43에서 300분할 기준으로 수정되었다.
- 현재는 다시 100분할, 비레버리지 기준으로 운영 중이다.

2. v53 baseline 해석 주의
- baseline이 이전 베스트 후보를 그대로 물려받은 기준선이 아니다.
- behavioral guard 조합의 실제 성과에 집중해야 한다.

3. v54/v55 러너의 실제 사용 필드 주의
- diverse entry modes 러너는 behavioral guard 계열 필드를 직접 다 쓰지 않는다.
- 따라서 base config에 들어 있는 max_active_cap, loss_streak 계열이 baseline 동작에 반영되지 않을 수 있다.
- 이 mismatch는 추후 정리 필요.

4. 파일명과 실제 동작 불일치 가능성 주의
- 예: long_div_baseline_lossbrake 같은 이름은 문자 그대로 해석하면 안 될 수 있다.
- 반드시 해당 러너 코드를 기준으로 실제 사용 필드를 점검해야 한다.

실험 설계 원칙
1. 먼저 전용 전략 개별 완성
- 숏 전용
- 롱 전용
2. 그 후 MDD 10% 미만 후보군만 별도 관리
3. 이후 그 안에서 cd_value 순 비교
4. 마지막에 결합 또는 레버리지 검토

현재 전용 전략 후보군에서 읽어야 하는 포인트
숏 전용
- 메인형 수익 보존
- 극방어형 MDD 최소화
- dense skip, quiet first, reentry gap 같은 진입 모드형의 재현성

롱 전용
- behavioral guard 계열의 강한 효과
- confirmation + reentry 철학의 재현성
- MDD 10% 미만을 유지하면서 수익을 조금씩 올릴 수 있는지

다음 어시스턴트가 반복하지 말아야 할 실수
1. 최근 사용자가 싫어한 미세 파라미터 조정으로 다시 회귀하지 말 것
2. 양방향 전략 하나로 바로 해결하려 하지 말 것
3. baseline 이름만 보고 실제 동작을 오해하지 말 것
4. 레버리지를 다시 검토하더라도, 저MDD 개별 전용 전략군 정리 이후에 할 것
5. 결과 보고 시 사용자가 원하는 항목 중 빠뜨리는 것이 없도록 할 것

추천 유지 문장 습관
- 이번 라운드는 어떤 철학을 시험했는지 먼저 말한다.
- 수익형과 극방어형을 구분해서 말한다.
- MDD 10% 미만 여부를 먼저 확인한다.
- baseline과 새 후보를 비교할 때, 수익 감소 대비 MDD 개선 폭을 함께 본다.

현재 요약
- 이 프로젝트는 이제 단순한 조건 튜닝이 아니라, 서로 다른 운용 철학을 병렬적으로 시험하는 단계다.
- 다음 어시스턴트는 반드시 실험 철학의 차이를 먼저 설명하고, 그 위에서 후보를 비교해야 한다.
