백테스트 핵심 전제

이 문서는 최근 기준선 파일들에서 반복적으로 확인된 핵심 전제를 정리한 것이다.
세부 파라미터는 버전별로 다르지만, 아래 항목들은 비교 기준으로 계속 중요하다.

1. 기본 자산
- initial_asset = 100.0

2. 기본 자산 분할
- position_fraction = 0.01
- 해석: 진입 1회당 기본 자산의 1%를 사용한다는 기준선이 최근 베이스 파일들에서 반복 확인됨

3. 수수료
- fee_per_side = 0.0004
- 해석: 편도 0.04% 수수료를 반영

4. 진입 방식
- entry_on_next_bar_open = true
- 해석: 신호가 난 직후 같은 봉 체결이 아니라 다음 봉 시가 진입 기준

5. 데이터/최소 준비 구간
- min_bars = 120
- data_dir = 5m_data
- 해석: 5분봉 데이터와 최소 준비 구간을 기준으로 실험이 구성됨

6. 롱/숏 허용
- allow_long, allow_short는 전략별로 분리
- 롱 베이스는 allow_long = true, allow_short = false
- 숏 베이스는 allow_long = false, allow_short = true

7. 최근 베이스에서 자주 보인 리스크/거버넌스 항목
- atr_stop_mult
- rr_mult
- timeout_bars
- fail_fast_bars
- fail_fast_min_progress_r
- protect_mode = time_reduce
- time_reduce_bars
- time_reduce_to_risk_frac
- loss_streak_trigger_losses
- loss_streak_window
- loss_streak_freeze_steps
- dd_brake_trigger_pct
- dd_brake_freeze_steps

8. 최근 베이스에서 자주 보인 거래 제어 항목
- top_n_per_timestamp = null
- top_n_long_per_timestamp = null
- top_n_short_per_timestamp = null
- score_min_long / score_min_short
- score_dev_weight / score_rsi_weight / score_wick_weight
- max_active_cap
- max_per_symbol
- reentry_gap_after_close_bars
- entry_gap_same_symbol_bars
- require_prior_signals_same_symbol
- prior_signal_window_bars
- first_signal_after_quiet_bars
- allowed_entry_hours_utc
- skip_dense_ts_threshold

9. cd_value 정의
- cd_value = 100 × (1 - |mdd_pct|/100) × (1 + final_return_pct/100)

10. 현재 프로젝트의 실전 평가 기준
- 저장소 전체 결과 기준
- MDD 5% 미만 후보만 남긴다
- 그 안에서 cd_value 1위를 본다
- 즉 수익률 단독이 아니라 MDD를 먼저 반영한 뒤 수익을 본다

11. 보조 지표
- trades
- win_rate_pct
- pf
- max_conc
- same_bar_trades
