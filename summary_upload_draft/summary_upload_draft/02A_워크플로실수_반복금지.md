워크플로 실수 반복 금지

1. LFS pointer 문제
- 에러 예시: ValueError: LFS pointer
- 의미: runner가 실제 데이터 파일 대신 LFS 포인터 텍스트를 읽은 상태일 수 있다.
- 금지: 데이터 체크 없이 바로 실행하기
- 점검: 데이터 파일이 실제 원본 내용인지 확인하고 실행

2. 결과 확인 경로 부재
- 금지: 실행 후 결과를 즉시 확인할 수 없는 구조로 두기
- 원칙: 결과 파일 위치, 요약 파일, 정렬 기준이 바로 보이도록 유지

3. 무거운 저장소 쓰기
- 금지: 여러 파일을 한 번에 생성/수정
- 원칙: 저장소 쓰기가 흔들릴 때는 파일 하나 단위로 반영
- 현재 운영 대안: 초안은 로컬 파일로 받고 사용자가 직접 업로드

4. 질문 해석 오류
- 금지: 직전 Q1이 무엇이었는지 바뀌어서 진행하기
- 원칙: 사용자가 지목한 기준 질문과 현재 작업 범위를 먼저 고정
