# 프로젝트 목표와 절대 제약

## 프로젝트명

자동매매 전략 개발 3

## 최종 목적

목표는 단 하나다.
자동매매로 장기적으로 돈 버는 전략을 찾는 것.

## 평가 철학

다음 해석은 금지한다.

- 승률 중심 판단
- RR 중심 판단
- 개별 종목 성과 중심 판단
- 부분 표본 성과를 전체 성과처럼 해석

판단은 오직 포트폴리오 순차 시뮬 기준으로 한다.
핵심은 EV > 0 구조이며, 최소 통과 조건은 `최종 수익률 > MDD`다.

## 최종 판단 지표

반드시 아래 값으로만 판단한다.

- 최종 자산
- 최종 수익률
- MDD
- Peak 또는 백테스트 중 최대 자산 수익률
- 거래수
- 최대 동시 포지션
- 총 승률
- 롱 승률
- 숏 승률

## 절대 데이터 제약

사용 데이터는 오직 OHLCV.

금지 데이터:
- OI
- 오더북
- 체결 데이터
- 뉴스
- 외부 데이터

## 타임프레임 제약

- 5분봉 only

타임프레임 확장 금지.

## 청산 제약

허용:
- 고정 SL
- 고정 TP

금지:
- 트레일링
- BE
- 조건부 청산
- 시간 청산
- adaptive TP/SL

## 사용자가 싫어하는 것

- 중간 보고
- 장황한 설명
- 미확정 결과 공유
- 나중에 무효화될 수 있는 수치 먼저 제시
- 같은 질문 반복
- 실패한 축 재제안
- 장난감 코드
- 타임프레임 확장
- 승률 중심 해석

## 사용자가 선호하는 것

- 결론 먼저
- 실제 완주 결과만
- 숫자 중심
- 전수 검증 우선
- 안 되면 바로 폐기 후 다음 조건으로 이동

## 다시 제안 금지 축

이전 세션에서 반복적으로 실패하거나 금지된 축:

- 추세추종
- 구조 분석 놀이
- 패턴놀이
- 세션/시간 필터
- 변동성 압축
- OI 기반 아이디어
- 상태 분류 엔진
- fractal
- adaptive TP/SL
- minor washout
- event naming 기반 스캐너

## 전략 철학

관심 없는 것:
- 평범한 상태
- 미세한 잡신호
- 승률 장난
- 복잡한 필터 누적
- 과최적화 구조

집중할 것:
- 비정상 상태
- 명확한 실패
- 짧은 SL
- 명확한 TP
- 비정상 → 실패 → 반응 구조

## 매우 중요한 운영 판단

다음 세션에서 부분 표본 결과, 별칭만 남은 전략, 전수 미검증 결과는 모두 무효로 취급해야 한다.
샘플 20종목은 조건 탐색용이다. 검증은 반드시 전체 CSV 전수 기준이어야 한다.
