## 기본 데이터 처리 원칙

1. zip이면 압축 해제
2. 내부 CSV 전부 읽기
3. 종목명은 파일명으로 구분
4. timestamp 기준 정렬
5. 중복 제거
6. 통합 데이터셋 구축

## 중복 처리 원칙

- 같은 파일명 + 같은 내용이면 완전 중복 제거
- 다른 파일명이어도 timestamp/open/high/low/close/volume 전체가 동일하면 중복 가능성 점검
- 같은 종목의 기간이 일부 겹치면 timestamp 기준 dedup
- 중복 제거 내역은 로그로 짧게 남길 것

## 대표 20종목 샘플 운용

사용자는 전체 373개 이상 전수를 매번 다 돌리면 조합 탐색 횟수가 너무 적어진다고 판단했다.
따라서 다음 루프를 선호한다.

1. 대표 20종목 샘플에서 조건 조합 탐색
2. 샘플 1위만 전체 CSV 전수 검증
3. 전수 검증에서 `최종 수익률 > MDD`가 아니면 즉시 폐기
4. 다시 샘플 탐색 복귀
