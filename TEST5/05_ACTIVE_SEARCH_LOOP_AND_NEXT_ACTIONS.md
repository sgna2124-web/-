# 현재 유효한 탐색 루프와 다음 액션

## 현재 유효한 운영 루프

사용자가 최종적으로 선호한 루프는 아래 구조다.

1. 대표 20종목 샘플에서 조건 조합 탐색
2. 샘플 상위 1위만 선택
3. 그 1위만 전체 CSV 전수 검증
4. 전체 검증에서 `최종 수익률 > MDD`가 아니면 즉시 폐기
5. 즉시 다시 대표 20종목 샘플로 돌아가 새로운 조건군 재탐색

## 매우 중요한 해석

- 샘플 단계는 탐색 단계다
- 전체 전수 단계만 검증 단계다
- 전체 검증 탈락 시 같은 조합을 붙잡지 말 것
- 즉시 다음 조건군으로 넘어갈 것

## 조건 탐색 방향

이전에는 아래 축의 미세 변형만 과도하게 반복한 문제가 있었다.

- 비정상 확장 실패 복귀
- wick reclaim
- lookback 강화
- delay reclaim
- open/midpoint/prior level reclaim

다음 세션에서는 이 축의 숫자만 흔드는 파라미터 탐색 비중을 줄이고, 구조 자체를 넓혀야 한다.

## 새로 포함해야 할 구조군

1. 이벤트 이후 비확장 정지 구조
2. 연속 봉 전달 단절 구조
3. 돌파는 났지만 종가가 못 따라가는 구조
4. 이전 스윙 회수 실패 구조
5. 비정상 거래량 대비 가격 전달 부족 구조
6. 상하 비대칭 캔들 군집 구조
7. 단일 이벤트가 아니라 2~4봉 묶음 실패 구조
8. 고점/저점 갱신 후 즉시 복귀가 아니라 무효화 누적 구조
9. 밴드 이탈 후 전달 실패 구조
10. 모멘텀 극단 후 꺾임 구조
11. 거래량 이상값 + 가격 비반응 구조
12. 변동성 확장 + 방향 붕괴 구조

## 지표 기반 탐색도 필수

사용자가 최근 직접 요구했다.
"파라미터만 바꾸지 말고 여러 지표 조합도 해봐"

따라서 다음 지표들을 포함한 구조 탐색을 반드시 병행할 것.

- RSI
- Stoch 또는 Stoch RSI
- CCI
- MFI
- ATR
- Bollinger Band
- Keltner Channel
- Donchian
- ADX
- ROC
- VWAP 괴리
- OBV 기울기
- CMF

## 지표 조합 설계 예시

- 과열/과매도 + 전달 실패 + 복귀
- 변동성 확장 + 방향 붕괴
- 밴드 이탈 + 복귀
- 거래량 비정상 + 가격 미반응
- 모멘텀 극단 + 꺾임

## 다음 세션의 실전 순서

1. 실제 업로드 파일 재집계
2. 엔진 검증 재확인
3. 대표 20종목 샘플 재구성
4. 새 구조군 + 지표 조합군으로 샘플 탐색
5. 샘플 1위만 전체 전수 검증
6. 탈락 시 즉시 폐기 후 4로 복귀
7. 통과 시 실제 전략 정의를 파일로 저장

## 전략 저장 원칙

다음 세션에서는 절대 별칭만 남기지 말 것.
반드시 아래를 저장할 것.

- 전략 이름
- 방향
- 사용 지표 또는 이벤트 구조
- 진입 조건
- 실패 조건
- 복귀 기준
- 진입 시점
- SL 규칙
- TP 규칙
- lookback
- range/body/volume/wick 또는 지표 임계값
- delay 조건
- 최대 허용 SL 폭이 있으면 그것도

## 현재 시점에서의 최종 한 줄

좋아 보였던 여러 결과는 있었지만, 검증 가능한 전략 정의와 전체 전수 통과가 함께 남은 확정 전략은 아직 없다.
