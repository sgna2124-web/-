# 백테스트 엔진 규칙

이 파일은 결과의 유효/무효를 가르는 핵심 규칙이다.
하나라도 어기면 결과는 무효다.

## 포지션 sizing
- 포지션은 무조건 자산의 1%만 진입
- 각 포지션 진입 순간의 계좌 자산 기준 1%
- 진입 순간 스냅샷을 떠서 포지션 명목금액 고정
- 청산 전까지 변하면 안 됨

## 수수료
- 편도 0.04%
- 왕복 0.08%
- 진입과 청산 각각 반영

## 동일봉 처리
- 동일 봉에서 SL/TP가 동시에 터치되면 무조건 SL 우선

## 진입 금지 조건
- TP 예상 수익폭이 0.3% 미만이면 진입 금지

## 포트폴리오 순차 시뮬
- 종목별 개별 성과로 최종 판단하지 않음
- 모든 종목의 신호를 하나로 합쳐 단일 계좌 기준으로 처리
- 절차: 신호 생성 → 이벤트 병합 → 시계열 정렬 → 단일 계좌 순차 처리

## 정렬 기준
1. timestamp 오름차순
2. 같은 timestamp면 symbol 오름차순
3. 그래도 같으면 원시 인덱스 순

## 재현성
- 같은 입력이면 같은 결과
- 최소 2회 이상 재실행해서 결과가 완전히 같아야 함

## 즉시 폐기 조건
- NaN
- inf
- 비정상 손익
- 비정상 포지션 크기
- 논리적 RR 범위를 넘는 단일 트레이드 손익
- 재실행 시 결과 불일치

## 체크리스트
- 입력 데이터 NaN 여부
- timestamp 정렬 일관성
- 진입 금액이 진입 시점 자산의 정확히 1%인지
- 포지션 오픈 후 명목금액 고정되는지
- 동일봉 SL/TP 동시터치 시 SL 우선 처리되는지
- TP 예상 수익폭 0.3% 미만 차단되는지
- 수수료 편도 0.04%가 진입/청산 각각 반영되는지
- Peak/MDD 계산 일관성
- 같은 입력으로 2회 실행 시 완전히 같은지

## 결론
과거 플러스 결과 일부는 이 엔진 규칙을 어겨서 무효화되었다.
다음 세션에서는 엔진 검증 통과 전 결과를 믿으면 안 된다.
