# 새 대화창 시작 문장과 체크리스트

## 짧은 시작 문장

이전 프로젝트를 이어간다. 먼저 현재 업로드된 전체 zip/CSV를 다시 집계하고 중복 제거 후 엔진 검증부터 해라. 이후 대표 20종목 샘플에서 조건 조합 탐색을 하고, 샘플 1위만 전체 CSV 전수 검증해라. 전체 검증에서 최종 수익률 > MDD가 아니면 즉시 폐기 후 다시 샘플 탐색으로 돌아가라. 부분 결과와 별칭만 남은 전략은 전부 무효다. 이번부터는 파라미터만 바꾸지 말고 RSI, Stoch, CCI, MFI, ATR, BB, Keltner, Donchian, ADX, ROC, VWAP 괴리, OBV, CMF 등 OHLCV 기반 지표 조합도 함께 탐색해라.

## 자세한 운영 문장

이 프로젝트는 5분봉 OHLCV만 사용해서 자동매매 전략을 찾는 작업이다. 모든 백테스트는 포트폴리오 순차 시뮬, 진입당 자산 1% 고정, 수수료 편도 0.04%, 동일봉 SL 우선, TP 예상폭 0.3% 미만 진입 금지, 재현성 검증 필수라는 규칙 아래 수행해야 한다. 조건 탐색은 대표 20종목 샘플에서 밀도 있게 하고, 샘플 1위만 전체 CSV 전수 검증한다. 전체 검증에서 최종 수익률 > MDD를 만족하지 못하면 즉시 폐기하고 다시 샘플 탐색으로 돌아간다. 부분 결과, 별칭만 남은 전략, 전수 미검증 결과는 모두 무효다.

## 다음 세션 첫 작업 체크리스트

- [ ] 실제 업로드 파일 목록 다시 확인
- [ ] zip 압축 해제
- [ ] CSV 전체 목록, 종목 수, 기간, 중복 여부 재집계
- [ ] 대표 20종목 샘플 구성 재확정
- [ ] 엔진 검증부터 다시 고정
- [ ] 자산 1% 규칙 확인
- [ ] 동일봉 SL 우선 확인
- [ ] 수수료 확인
- [ ] TP 0.3% 미만 진입 금지 확인
- [ ] 재현성 2회 이상 확인
- [ ] 샘플 탐색 시작
- [ ] 샘플 1위만 전체 전수 검증
- [ ] 탈락 시 즉시 폐기 후 재탐색
- [ ] 통과 시 실제 규칙 정의 저장

## 보고 형식

사용자가 선호하는 보고 형식:

1. 결론 먼저
2. 전체 데이터 포함 여부 명시
3. 실제 완주한 조합 수 명시
4. 샘플 단계인지 전체 검증 단계인지 명시
5. 상위 결과는 짧고 숫자 중심으로만 제시
6. 전체 검증에서 탈락이면 즉시 폐기라고 말하고 다음 탐색으로 이동

## 금지 보고 형식

- 부분 CSV 결과를 전체처럼 말하기
- 전략 정의 없이 별칭만 남기기
- 미완주 상태에서 숫자 제시하기
- 실제 실행 수와 추정 수를 섞어서 말하기
- 과장된 반복 횟수 표현 사용하기

## 마지막 경고

이전 대화에서 나온 고수익 수치들 중 상당수는 지금 시점에서 검증 가능한 전략 정의가 없다. 따라서 그 수치를 재사용하지 말고 실제 파일과 실제 규칙 정의 기준으로 다시 시작해야 한다.
