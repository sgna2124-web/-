# TEST5 인덱스

이 폴더는 프로젝트 "자동매매 전략 개발 3"의 인수인계 전용 기록이다.
목적은 사람이 보기 좋게 꾸미는 것이 아니라, 다음 세션의 내가 빠르게 상태를 복원하고 같은 실수를 반복하지 않게 만드는 것이다.

## 파일 구성

- `00_INDEX.md`
  - 이 폴더의 목적과 파일 구조
- `01_PROJECT_OBJECTIVE_AND_NON_NEGOTIABLES.md`
  - 프로젝트 최종 목적, 절대 제약, 사용자 성향, 금지 축
- `02_BACKTEST_ENGINE_RULES.md`
  - 엔진 규칙, 포지션 sizing, 수수료, 동일봉 처리, 재현성 규칙
- `03_DATA_STATUS_AND_DEDUP_WORKFLOW.md`
  - 업로드 파일 상태, zip/CSV 처리, 중복 제거, 대표 20종목 샘플 운용 방식
- `04_SEARCH_HISTORY_AND_INVALIDATION_LOG.md`
  - 지금까지의 탐색 흐름, 무엇이 무효가 되었는지, 왜 무효가 되었는지
- `05_ACTIVE_SEARCH_LOOP_AND_NEXT_ACTIONS.md`
  - 다음 세션에서 바로 돌려야 할 루프, 조건 탐색 방식, 검증 순서
- `06_STARTER_PROMPTS_AND_CHECKLIST.md`
  - 새 대화창 시작용 문장, 체크리스트, 보고 형식

## 최우선 원칙

1. 부분 표본 결과를 최종처럼 다루지 말 것
2. 전략 별칭만 남기지 말고 실제 규칙 정의를 반드시 함께 저장할 것
3. 전체 CSV 전수 검증에서 `최종 수익률 > MDD`를 만족하지 못하면 즉시 폐기할 것
4. 사용자는 장황한 설명보다 실제 완주 결과를 선호한다
5. 다음 세션에서는 반드시 실제 업로드 파일 기준으로 데이터 수를 다시 집계할 것

## 현재 최종 상태 한 줄

지금 시점에서 전수 검증을 통과한 확정 전략은 없다. 샘플 20종목 탐색 결과 중 좋아 보였던 수치들은 전체 전수 검증, 전략 정의 소실, 부분 표본 문제 때문에 최종 채택 대상으로 인정되지 않는다.
