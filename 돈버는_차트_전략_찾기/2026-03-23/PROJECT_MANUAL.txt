프로젝트 이름
돈버는 차트 전략 찾기

프로젝트 최종 목표
목표는 단 하나다.
자동매매로 장기적으로 돈을 버는 전략을 만드는 것.

세부 목표
5분봉 기반 자동매매
바이낸스 선물 다종목 동시 운용
인간 심리 고려 없음
고빈도/저빈도 중요하지 않음
승률 자체가 목표 아님
RR 자체가 목표 아님
최종 기대값 EV가 양수면 됨

핵심 철학
카지노처럼 아주 작은 통계적 우위라도 반복해서 돈을 버는 구조를 찾는 것이 목표다.
하지만 여기서 말하는 작은 우위는 평범한 상태를 잘게 나눈 50.1% 같은 미세 우위가 아니다.
사용자는 비정상 상태, 선명한 진입 근거, 짧고 명확한 SL, 분명한 TP를 선호한다.
즉 평범한 상태를 잘게 쪼개는 전략보다, 비정상 상태에서 명확한 무효화 기준이 있는 사건 중심 전략을 선호한다.

절대 고정 제약
1. 데이터는 OHLCV만 사용
외부 지표, 온체인, 뉴스, 호가, 오더북, 체결흐름, 외부 구조 데이터 금지
OI는 이미 사용해봤고 실패했으니 다시 꺼내지 말 것

2. 기본 타임프레임은 5분봉
15분, 1시간, 큰 타임프레임으로 도망가는 제안 금지
사용자는 5분에서 실패한 미시구조 전략이 큰 타임프레임으로 간다고 성공한다고 보지 않는다

3. 청산은 고정 SL/TP 기반
추가 청산, BE 이동, 트레일링, 복잡한 조건부 청산은 기본 철학과 맞지 않음
손절과 익절은 진입 시점에 선명하게 결정되어야 함

4. 최종 평가는 다종목 순차 시뮬레이션 기준
개별 심볼 승률이 아니라 전체 포트폴리오 시간순 순차 시뮬 기준으로 판단
반드시 봐야 하는 것은:
최종 자산
MDD
최대 수익 종목
최대 손실 종목
최대 포지션 보유 수

5. 실제 백테스트 엔진 틀 유지
사용자가 준 파이썬 엔진의
엑셀 로드/저장
print
plt
시계열 순차 시뮬
결과 기록 구조
for문 구조
이런 틀은 유지해야 한다.
바꿔도 되는 것은 진입 조건, 상태 변수, 진입 시점의 고정 SL/TP 설정뿐이다.

매우 중요한 작업 규칙
1. 절대 장난감 백테스트 코드를 따로 만들지 말 것
단일 종목 미니 테스트 코드, 승률만 출력하는 코드, 폴더의 엑셀 아무거나 읽는 코드 금지
반드시 기존 엔진 틀을 유지한 채 전략 로직만 교체할 것

2. 결과는 승률만 보여주면 안 됨
반드시 최종 순차 수익, MDD, 최대 수익 종목, 최대 손실 종목, 최대 포지션 수 등 기존 엔진 결과 출력이 살아 있어야 함

3. 이미 실패한 축은 다시 제안하지 말 것
세션, 시간, 요일, 변동성 압축 돌파, 구조 해석, 일반적 추세 전략, 패턴 기반 전략, 이벤트 이름 기반 전략, OI 재활용, 타임프레임 키우면 될 것 같은 말 금지

4. 같은 실수 반복 금지
매뉴얼을 받고도 같은 전략축, 같은 제안 흐름, 같은 오류 반복 금지
특히 포기할 생각이 있냐, OHLCV 공간이 끝난 것 아니냐 같은 질문 반복 금지

5. 새 전략을 만들더라도 반드시 기존 엔진 구조에 삽입하는 형태로 제시할 것
필요 시 전체 코드를 채팅에 그대로 붙여 넣을 준비를 할 것
파일 링크가 안 열리는 경우가 잦았음

사용자의 핵심 관점
1. 5분봉에서 실패한 전략은 더 큰 타임프레임에서도 본질적으로 실패할 가능성이 높다
2. 수수료 문제는 핑계가 아니다. 이미 변동성 필터 등은 충분히 고려했다
3. 정상 상태에서의 미세 확률 우위에는 관심이 약하다
4. 비정상 상태, 극단 상태, 선명한 무효화 기준, 짧은 손절, 명확한 목표가 있는 구조를 더 신뢰한다
5. 이론적 명분보다 실제 백테스트 기대값이 더 중요하다
6. 돈만 벌면 된다. 말장난, 스토리텔링, 용어 바꾸기에는 관심 없다

10. 단일 종목 이벤트 이름 기반 접근

이미 진행했던 큰 흐름 요약
1단계
차트형/추세형/구조형 전략들 시도
결과: 과적합 또는 EV 부족으로 실패

2단계
Casino Probability Engine
상태 → 확률 → EV 구조
feature는 ATR percentile, momentum, wick skew 등
결과: 상태 공간이 너무 거칠고 신뢰도 부족, EV 음수

3단계
세션/시간/변동성 접근
결론: 방향 엣지보다 변동성 설명력만 약간 있었고 결국 다시 압축폭발/구조 루프로 회귀
실전적으로 큰 의미 없다고 판단

4단계
이벤트 스캐너 v1/v2
사람이 정의한 이벤트 이름을 대규모 스캔
결과: durable edge 없음
strict 후보 대부분 0 또는 우연 수준

5단계
Threshold Scanner
연속형 피처 임계값 조합 탐색
마이너 알트에서 약한 단서가 있었으나 전략화 후 전체 유니버스 재현 실패

6단계
Minor Washout / Symbol Fit / Symbol State Washout 변형
종목 분류 기준 문제
상태 필터가 전체 시장에 퍼짐
시뮬레이터/포지션 관리 버그성 이슈
전략 카드로는 폐기 수준

7단계
State Miner v1
가능한 많은 OHLCV 연속형 상태를 single/pair/triple 조합으로 광범위하게 스캔
BTC strict survivors = 0
다른 단일 알트도 strict survivors = 0
전체 종목 결과는 당시 진행 중
사용자는 이 결과가 0이면 5분 OHLCV-only 공간을 공식 종료할 의향이 있음

사용자가 직접 언급한 의미 있는 전략 힌트
볼린저밴드 사용
소스는 시가
길이 288
std 7
종가가 밴드 밖으로 이탈했다가 다시 복귀하면 역추세 진입

사용자 평가
MDD는 높지만 수익 기대치가 꽤 높았다
비정상적 이탈이라는 명분이 선명하다
역추세라서 SL이 짧고 기준이 명확하다
TP도 명확하다
즉 짧은 SL + 비정상 상태 + 명확한 무효화 철학이 중요하다

볼밴 전략 관련 추가 설명
사용자는 볼린저밴드 길이 288, std 7, 소스 시가를 사용하고 있었다.
볼밴 이탈 후 복귀할 때 진입한다.
볼밴 이탈을 비이성적인 폭발로 보고 평균 회귀를 노리는 것이다.
손절은 최근 고저점.
익절은 entry-sl의 2배 지점과 진입 시 288 sma 위치 중 진입가보다 더 먼 가격으로 설정한다.
그리고 288 sma를 반대로 넘기면 즉시 익절한다.
이건 하나의 예시일 뿐이고, 이런 식으로 명확한 규칙과 손익절 방식이 있는 전략을 선호한다.
이 반전 진입 전략은 손절이 짧고 익절을 길게 기대하는 전략이다.
추세 전략은 보통 여기까지는 확실히 갈 것이라는 기준이 애매하다고 느낀다.
그래서 이벤트 기반 반전/사건 전략에 더 끌린다.

직전까지 괜찮았던 철학
Failed Displacement Reclaim 계열
핵심 아이디어
최근 수용 범위를 비정상적으로 강하게 이탈
그러나 그 새 가격대를 시장이 즉시 받아들이지 못하고 다시 범위 안으로 reclaim
이 경우 반대 방향 진입

사용자 반응
오호 나쁘지 않은데 라고 반응했음
즉 이 방향은 최소한 사용자 철학과는 잘 맞았다.
하지만 실제 엔진 삽입 후 20개 종목 테스트에서는 손실, 승률 약 26% 수준으로 성과 실패.
즉 철학은 맞았지만 전략은 실패.
살아남은 것은 철학뿐이다.

후임이 절대 하면 안 되는 실수
1. 사용자가 준 엔진 틀을 무시하고 단일 종목용 장난감 코드를 만드는 것
2. 승률만 보여주는 것
3. 폴더의 모든 xlsx를 무차별 로드해서 컬럼 에러를 일으키는 것
4. 전략과 관계없는 환경 문제를 코드 문제처럼 섞는 것
5. 포기할 거냐, OHLCV 공간 끝난 것 아니냐 반복 질문
6. 이미 실패한 세션/시간/압축폭발/구조/OI 축을 다시 꺼내는 것
7. 타임프레임 키우면 성공할 수 있다는 관점 반복
8. 목표를 정상 상태의 미세 확률 우위로 오해하는 것
9. 상태 공간 확장이라는 명목으로 결국 같은 넓은 일반 탐색 반복
10. 매뉴얼을 받고도 같은 흐름을 또 밟는 것

응답 원칙
1. 사용자의 목표를 왜곡하지 말 것
목표는 돈 버는 전략이다.
전략이 아니라 상태지도 같은 식으로 본질을 바꾸지 말 것.

2. 사용자가 이미 시도한 과정을 존중할 것
초급 수준의 그건 해봤냐 질문 금지

3. 사용자가 불쾌해하는 포인트 피할 것
같은 질문 반복
같은 전략축 반복
명백히 실패한 축 재제안
핵심 작업 규칙 무시

4. 항상 엔진 기준으로 말할 것
새 전략이든 새 시도든
기존 엔진 틀에 진입 조건과 손익절 부분만 바꿔 넣는 관점으로만 접근할 것

5. 전체 결과 중심으로 판단할 것
승률 중심 사고 금지
최종 자산, MDD, 종목별 손익, 최대 포지션, 분포를 볼 것

state_miner 결과가 나왔을 때 행동 규칙
1. strict survivors가 0이면
쓸데없는 희망고문 금지
5분 OHLCV-only 공간 종료 판단을 도와야 한다

2. strict survivors가 1~2개면
새 철학 제안보다 기존 엔진에 해당 상태를 진입 로직으로 넣는 방향으로 가야 한다

3. robust하게 나오면
그때 포지션 sizing이나 포트폴리오 구조는 나중 문제
지금은 전략 존재 여부가 먼저

4. 결과 해석은 단순해야 함
계속할지 종료할지
전략화할지 말지
이 수준으로 빠르게 판정

새 전략을 제안할 때의 원칙
1. 사용자 철학과 맞아야 한다
비정상 상태
선명한 무효화
짧은 SL
명확한 TP
사건 중심 진입

2. 기존 실패 축과 달라야 한다
세션, 시간, 압축폭발, 구조, 일반 추세, 일반 상태분류, OI 반복 금지

3. 살아 있는 힌트를 중심으로 가야 한다

현재 시점의 가장 정확한 한 줄 요약
5분 OHLCV-only 공간에서 일반적 상태/패턴/이벤트/추세 전략은 대부분 실패했다.
남은 가능성은 매우 좁고, 비정상 상태에서 명확한 무효화와 짧은 손절을 갖는 사건 중심 전략 쪽에 있다.
다만 이벤트 직후 1~2봉 반응만으로는 부족할 수 있고, 이벤트 이전 문맥과 이벤트 이후 10~20봉 소화 구조를 함께 기계적으로 정의하는 방향이 다음 후보가 될 수 있다.

새 대화창에서 원하는 태도
1. 위 내용을 이미 완전히 이해한 상태로 시작할 것
2. 같은 실패축을 다시 꺼내지 말 것
3. 같은 질문을 반복하지 말 것
4. 목표를 돈 버는 전략이라는 본질에서 벗어나게 재정의하지 말 것
5. 새 전략을 제안한다면 바로 엔진 삽입 가능한 형태 또는 최소한 파인스크립트로 빠르게 검증 가능한 형태로 제시할 것
6. 결과가 애매하면 희망고문 말고 빨리 버릴 것
7. 좋은 전략을 떠올리는 데 집중할 것. 쓸데없는 질문으로 대화 길게 끌지 말 것
