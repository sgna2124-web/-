어시스턴트 응답 및 코드 산출 행동 수칙

이 문서는 새 대화창의 어시스턴트가 반드시 적용해야 하는 응답 방식과 파이썬 코드 산출 규칙이다.
CORE_SUMMARY/09_NEXT_CHAT_MANUAL.md 와 함께 적용한다.

1. 응답 방식
대답은 감정적 공감보다 핵심 전달을 우선한다.
불필요한 위로, 변명, 장황한 배경 설명을 줄인다.
사용자가 저장소 확인, 결과 분석, 코드 작성, 문서 갱신을 지시하면 다음 네 가지만 우선 보고한다.
1) 확인한 대상
2) 판단한 핵심 결론
3) 실제 수정 또는 생성한 파일
4) 다음 실행에 필요한 최소 지침

2. 파이썬 코드 산출 기본 원칙
파이썬 코드를 작성한 경우, 사용자가 특별히 다른 방식을 요청하지 않는 한 현재 대화창에 .py 파일로 만들어 업로드한다.
긴 코드는 채팅 본문에 전부 붙여넣지 말고, 파일로 제공한 뒤 본문에는 실행 명령과 핵심 변경점만 적는다.
저장소에 바로 커밋하는 것은 사용자가 저장소 반영을 명시했을 때만 수행한다.

3. 업로드 전 코드 점검 원칙
파이썬 파일을 업로드하기 전에 전체적으로 오류 가능성을 점검한다.
최소 점검 항목은 다음과 같다.
1) py_compile 수준 문법 오류
2) import 누락
3) 함수명, 변수명, 클래스명 오타
4) 정의되지 않은 FeaturePack 필드 참조
5) 정의되지 않은 StrategyParams 필드 참조
6) 정의되지 않은 entry_signal 또는 mask 참조
7) pandas/numpy 배열의 truth value ambiguity 가능성
8) UnboundLocalError 가능성이 있는 조건부 변수 할당
9) 사용자 로컬 경로, 결과 폴더명, 저장 파일명 불일치
10) 기존 백테스트 파일의 고정 환경 훼손 여부

4. 실제 실행 검증 기준
가능하면 업로드 전 py_compile을 통과시킨다.
외부 데이터, 사용자 로컬 경로, 대용량 OHLCV 파일이 없어 전체 백테스트 실행이 불가능한 경우에도 문법 검사와 정적 참조 점검은 수행한다.
실행 검증을 못 한 부분이 있으면 무엇을 못 했는지만 짧게 명시하고, 검증 가능한 오류는 사전에 제거한다.

5. 이 프로젝트에서 특히 금지할 반복 오류
정의되지 않은 로컬 변수 사용 금지.
numpy/pandas 배열을 if 조건에 그대로 넣는 구조 금지.
FeaturePack에 없는 속성을 전략 조건에서 호출하는 구조 금지.
StrategyParams에 없는 필드를 전략 로직에서 참조하는 구조 금지.
기존 공식 백테스트 환경인 자산 진입 비중 1%, 수수료 0.04%, 전체 종목 풀 기준을 코드 생성 과정에서 임의로 바꾸는 행위 금지.
