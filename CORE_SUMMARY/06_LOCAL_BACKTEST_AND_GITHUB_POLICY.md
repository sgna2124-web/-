# LOCAL BACKTEST AND GITHUB POLICY

## 현재 대전제
GitHub 저장소는 현재 LFS 예산 문제로 인해 대용량 데이터 기반 백테스트 실행 장소로 사용하기 어렵다.
따라서 당분간 다음 구조로 운영한다.

## 운영 구조
- 백테스트 실행: 사용자 로컬 컴퓨터의 Python 환경
- GitHub 저장소: 전략 코드, 결과 요약, 핵심 로그, 분석 문서 업로드 장소

## 이 방식의 장점
- LFS 예산 문제로 인해 Actions가 막혀도 연구 자체는 계속 가능
- 전략 코드와 결과 문서를 버전 관리할 수 있음
- 장기적으로는 저장소가 '실험 기록소 + 전략 지식 베이스' 역할을 하게 됨

## GitHub에 올릴 것
- 전략 코드
- 실험 설정 JSON
- 결과 요약 CSV/MD/TXT
- 핵심 수치만 담은 경량 산출물
- 분석 문서
- 인수인계 문서

## GitHub에 신중히 올릴 것
- 대용량 원시 데이터
- LFS를 강하게 소모하는 파일
- 로컬에서만 필요한 중간 산출물 전체

## 권장 방식
- 원시 OHLCV 대용량 파일은 로컬 보관 우선
- 저장소에는 재현에 필요한 최소 경량 결과와 코드만 업로드
- 필요시 결과 요약본과 핵심 통계만 남김

## 새 실험 업로드 체크리스트
1. 코드 파일명/버전명 정리
2. 결과 요약 파일 생성
3. baseline 대비 비교표 생성
4. 전략 카드 등록
5. 핵심 배운 점 기록

## 장기 목표
- 저장소를 '바로 실행되는 백테스트 플랫폼'이 아니라
  '전략 탐색의 누적 기억 장치'로 운영한다.
- 추후 LFS/데이터 정책이 정리되면 실행 자동화는 다시 붙일 수 있다.
