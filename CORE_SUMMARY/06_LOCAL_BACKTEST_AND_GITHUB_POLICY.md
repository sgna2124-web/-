로컬 백테스트 운영 원칙

공식 백테스트 조건
자산 진입 비중은 1% 고정
수수료는 0.04% 고정
FAST 예비검사는 사용하지 않음
공식 결과는 전체 종목 풀 백테스트 기준
조기탈락 규칙 사용 안 함
거래 로그 파일 생성은 필수 아님

결과 출력 최소 항목
strategy_name
side
result_path
trades
final_return_pct
max_return_pct
max_drawdown_pct
cd_value
verdict

업로드 원칙
대용량 csv는 업로드하지 않는다.
요약 json, master_summary.txt, 배치 registry 등 비교에 필요한 최소 결과만 업로드한다.
100MB를 넘길 수 있는 파일은 만들지 않거나 업로드 대상에서 제외한다.

현재 시간 단축 구조
백테스트 시간 단축은 현재 프로젝트의 중요한 성과다.
새 대화창의 어시스턴트는 이 구조를 유지해야 한다.
현재 구조의 핵심은 다음과 같다.
지표는 심볼 단위로 한 번 계산한 뒤 재사용한다.
내부 루프에서는 pandas 연산보다 배열 기반 연산을 우선한다.
거래별 상세 로그는 만들지 않고 요약 결과만 저장한다.
공식 판정에 불필요한 부가 출력은 줄인다.
전체 종목 풀 백테스트를 전제로 하되, 결과 저장은 가볍게 유지한다.

GitHub 반영 원칙
사용자는 로컬에서 실행 후 결과를 업로드한다.
어시스턴트는 업로드된 결과를 바탕으로 문서를 갱신한다.
워크플로우 기반 백테스트는 현재 핵심 운영 방식이 아니다.
