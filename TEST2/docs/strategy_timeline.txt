[전략/코드 타임라인 요약]

초기 목표
- OHLCV만으로 1500캔들 내 상태 진단 후 종목별/상태별 전략 카드 선택.
- 파인스크립트로 1차 검증 후 파이썬 포트폴리오(시간순 재순환) 백테스트로 확장.

Pine 단계(요약)
- Regime 기반 3모드(TrendPullback / SweepFade / Hybrid 등) 시도 → 다종목 장기에서 붕괴가 많아 Trend 중심으로 정리.
- TrendPullback 4.0~4.4: 풀백 2단계(발생→확인), 고정 SL/TP, 비용 게이트, 퍼센타일 레짐(1500).
- BE(1R/0.8R) 포함 버전(4.1/4.2) 실험 → 대부분 종목에서 총수익 저하로 BE 폐기.
- 헤드룸/꼬리/쇼크 필터(4.3/4.4) 실험 → 거래 감소로 손실 감소는 있었으나 총수익이 깎이는 경향.
- 결론: 5m에서 TrendPullback 계열로 ‘확실한 플러스 기대값’ 만들기 어려움.

Python 단계(학습형엔진 틀 유지)
- 원본 틀은 유지(엑셀/print/plt/순차 시뮬/루프/경로/형식).
- 수정은 진입 조건/상태 변수/SLTP만.
- 모드 3종:
  - v42: regime_ok 통과 시 진입
  - v42N: strong_ok 통과 시만 진입
  - mix: strong_ok면 공격, 아니면 방어 파라미터(진입만 엄격)
- 결과:
  - v42/mix: 동시 포지션 과다(>100)로 구조적 붕괴(MDD -80~-95%대)
  - v42N: 포지션 수/ MDD는 줄었으나 최종 자산 100 회복 실패(90대)
- A안: MIN_STOP_PCT_FOR_COST(최소 손절폭%) 추가 → MDD/동시보유는 크게 감소했으나 최종 100 안정적 회복 실패.

최종 결론
- 5m + 고정 SL/TP + TrendPullback 진입 정의 조합은 수수료 환경에서 기대값이 얇거나 음수로 판단되어 ‘5m 실패’로 기록.
