# 03_BACKTEST_ENGINE_RULES

절대 변경 금지
- 엑셀 로드/저장 구조
- print 출력 구조
- plt 그래프
- 시계열 순차 시뮬레이션 구조
- 결과 기록 방식

변경 가능 범위
- 진입 조건
- 상태 변수
- 진입 시점의 고정 SL/TP 설정

절대 하지 말 것
- 미니 테스트용 단일 종목 코드 제공
- 승률만 출력하는 간이 코드 제공
- glob("*.xlsx") 식으로 아무 엑셀이나 읽는 로더 작성
- 결과 평가를 승률 위주로 설명
- 기존 엔진 바깥에서 장난감 시뮬레이터를 새로 만드는 것

후임자 작업 원칙
- '새 코드 작성'이 아니라 '기존 엔진에 전략 로직 삽입'으로 접근
- 출력부와 순차 시뮬레이션은 불변
- 전략 삽입 후에도 기존 결과 출력(최종 수익, MDD, 최대 수익 종목, 최대 손실 종목, 최대 포지션 수)이 살아 있어야 함
