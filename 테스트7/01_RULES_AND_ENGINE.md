# 프로젝트 목적과 엔진 규칙

프로젝트 최종 목적
- 목표는 단 하나다.
- 자동매매로 장기적으로 돈 버는 전략을 찾는 것.

핵심 평가 철학
- 승률 중심 판단 금지
- RR 중심 판단 금지
- 개별 종목 성과 중심 판단 금지
- 오직 포트폴리오 순차 시뮬 기준의 EV > 0 구조만 본다

최종 판단 지표
- 최종 자산
- 최종 수익률
- MDD
- Peak 또는 백테스트 중 최대 자산 수익률
- 거래수
- 최대 동시 포지션
- 총 승률
- 롱 승률
- 숏 승률

최소 통과 조건
- 최종 수익률 > MDD

절대 제약
- 데이터는 OHLCV only
- 사용 가능 데이터는 오직 OHLCV
- 금지 데이터: OI, 오더북, 체결, 뉴스, 외부 데이터
- 타임프레임: 5분봉 only

청산 규칙
원본 운영 규칙 기준
- 고정 SL / TP only
- 금지: 트레일링, BE, 조건부 청산, 시간 청산, 적응형 TP/SL

중요한 충돌 사항
- 후반부 PineScript 러프 실험 단계에서는 사용자가 고정 SL/TP 방식에 강한 불만을 표했고, Pine 시도 코드에서는 종목/상태 적응형처럼 보이도록 유동 SL/TP 방향으로 바뀌었다.
- 그러나 프로젝트의 원래 파이썬 전수 엔진 규칙은 고정 SL/TP only였다.
- 다음 세션에서는 공식 백테스트 엔진 규칙과 Pine 아이디어 스케치를 분리해 관리해야 한다.

가장 중요한 엔진 규칙
- 포지션은 무조건 자산의 1%만 진입
- 정확한 의미: 각 포지션 진입 순간의 계좌 자산 기준 1%, 그 순간 스냅샷을 떠서 그 포지션 명목금액을 고정, 청산 전까지 변하면 안 됨
- 동일 봉에서 SL/TP 동시 터치 시 무조건 SL 우선
- 수수료 반드시 반영: 편도 0.04%, 왕복 0.08%
- TP 예상 수익폭이 0.3% 미만이면 진입 금지
- 포트폴리오 순차 시뮬 필수: 모든 종목 신호를 하나로 합쳐 시계열 순으로 정렬 후 단일 계좌 기준으로 처리
- 정렬 기준 고정: timestamp 오름차순, 같은 timestamp면 symbol 오름차순, 그래도 같으면 원시 인덱스 순
- 동일 입력이면 동일 결과
- 재현성 필수
- Peak / MDD 계산 일관성 유지
- NaN / inf / 비정상 손익 / 비정상 포지션 발생 시 즉시 폐기
- 단일 트레이드 수익률은 논리적 RR 범위를 절대 넘으면 안 됨

엔진 검증 체크리스트
- 입력 데이터 NaN 여부
- timestamp 정렬 일관성
- 동일 timestamp 처리 순서
- 포지션 진입 금액이 진입 시점 자산의 정확히 1%인지
- 포지션 오픈 후 명목금액이 고정되는지
- 동일봉 SL/TP 동시터치 시 SL 우선 처리되는지
- TP 예상 수익폭 0.3% 미만은 진입 차단되는지
- 수수료 편도 0.04%가 진입/청산 각각 반영되는지
- Peak/MDD 계산 일관성
- 같은 입력으로 2회 이상 실행 시 완전히 같은 결과인지
- NaN/inf/비정상 수익률/비정상 포지션 없는지

사용자 보고 성향 요약
- 중간 보고 싫어함
- 장황한 설명 싫어함
- 결론 먼저 선호
- 숫자 중심 선호
- 미확정 결과 공유 금지
- 부분표본 착시와 과장된 실행 횟수 보고를 매우 싫어함
