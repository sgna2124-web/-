# Pine 메모

이번 대화에서 Pine으로 시도한 방향
- 상태 기반 롱, 숏, 횡보 통합 프레임
- 롱 전용 강한 상승 추세와 눌림 반전
- 추세 점수, 지속성, 히스테리시스 강화
- n봉 전 대비 상대위치 기반 추세 정의
- range 또는 neutral 상태에서만 반전 진입
- 고정 SL TP 제거 후 유동 SL TP 시도
- 진입 위치를 시장가 대신 되돌림 지정가로 개선

사용자가 직접 지적한 규칙
- ta.sum 대신 math.sum을 쓰는 환경이라는 피드백
- adx 내장 호출 대신 DI와 ADX 직접 계산 필요
- 의미 없는 줄바꿈 금지
- if문이나 for문 안에 highest, lowest 금지
- 고정 SL TP를 박아두지 말고 종목과 상태에 맞게 유동적으로 생각할 것
- 진입 조건을 더 까다롭게 할 것
- 진입 위치와 SL, TP 설계가 별로라는 피드백 반복

핵심 교훈
- 코드 문법과 스타일 제약을 먼저 정확히 맞춰야 한다
- 추세 정의와 진입 정의를 한 덩어리로 섞지 말 것
- range 또는 neutral 상태 반전 전략은 아이디어로는 유효하지만, entry, SL, TP의 위치 선정이 아직 미숙하다
