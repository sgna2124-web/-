사용자의 최신 데이터 정책
- 백테스트 데이터 파일은 앞으로 계속 추가될 수 있음
- 새로운 데이터는 기존 데이터에 포함시켜 계속 백테스트해야 함
- 사용자가 보유한 종목 데이터는 약 600개라고 언급함
- 현재 대화에서 전달된 것은 그 일부에 불과함

데이터 처리 원칙
- zip 업로드 처리
- zip이면 압축 해제 후 내부 CSV 모두 읽기
- 종목명은 파일명으로 구분
- 같은 파일명 + 같은 내용이면 완전 중복 제거
- 파일명이 달라도 timestamp/open/high/low/close/volume 전체 동일 시 중복 가능성 점검
- 같은 종목의 기간 일부가 겹치면 timestamp 기준 dedup
- 중복 제거 내역은 짧게 기록

이번 대화에서 실제로 넓게 재시도된 방향들
1. 무차별 조합 탐색, 브루트포스
2. 상승 추세, 하락 추세, 횡보 상태 공통 전략 탐색
3. 상승 추세에서 하락 후 반전 롱
4. 하락 추세에서 상승 후 반전 숏
5. 횡보 반전
6. 상태 기반 전략 선택 프레임

반복적으로 시도된 구조 범주
- RSI 과열/과매도 + BB 이탈 + 복귀
- ADX 추세 중 모멘텀 꺾임
- VWAP 괴리 + 가격 미반응
- 거래량 스파이크 + 전달 실패
- 2~4봉 연속 방향 후 붕괴
- 밴드 이탈 + 복귀
- 거래량 비정상 + 가격 미반응
- 모멘텀 극단 + 꺾임
- 상승 추세 롱 전용 / 하락 추세 숏 전용 / 횡보 반전 전용
- 순수 캔들 구조
- EMA 정배열/기울기 기반 추세 판정
- n봉 전 대비 가격 상대위치 기반 추세 판정
- PineScript 상 range/neutral 상태에서만 반전 진입
- PineScript에서 유동 SL/TP, 지정가 되돌림 진입

특별히 기억할 교정 포인트
- 사용자는 상승 추세에서 하락 후 반전 시그널 롱을 원했지, 하락 중 칼날 잡기를 원하지 않았다.
- 고가 돌파까지 기다리기는 너무 늦다는 피드백이 있었다.
- 사용자는 하락 종료 + 초기 반전 시그널 위치를 원한다.
- 사용자는 고정 SL/TP 방식에도 강한 불만을 표했다.

사용자가 싫어하는 보고 방식
- 부분 CSV 결과를 전체처럼 말하기
- 대표 샘플 결과를 최종처럼 말하기
- 전략 정의 없이 별칭만 남기기
- 미완주 상태에서 숫자 제시하기
- 실제 실행 수와 추정 수를 섞기
- 과장된 실행 횟수 보고
